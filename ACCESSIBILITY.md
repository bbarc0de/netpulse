# Accessibility statement draft

> **Requires qualified legal and accessibility review.** This is a readiness statement, not a certification of conformance.

NetPulse aims to support keyboard navigation, visible focus, semantic labels and landmarks, readable contrast, reduced motion, responsive layouts, and accessible theme controls. Network charts and live measurement states should have textual equivalents; color must not be the only status signal.

Known review needs include full screen-reader testing, zoom/reflow at 200% and 400%, Windows high-contrast mode, chart data-table alternatives, Turnstile failure/accessibility behavior, and representative mobile assistive-technology testing. Report accessibility barriers through the repository contact route without including sensitive network data.
